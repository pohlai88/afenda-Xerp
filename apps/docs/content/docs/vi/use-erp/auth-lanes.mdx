## Overview

Afenda ERP groups authentication URLs into **lanes** — access, security, verify, recover, invite, and workspace. Use this map when you need the exact path for sign-in help or when pairing with the [Sign in guide](/docs/use-erp/sign-in).

<Callout type="info" title="Tenant URL">

Always use your organization's tenant host. Paths below are suffixes such as `/sign-in` on that host.

</Callout>

## Lanes

### Access auth lane

Đăng nhập, đăng ký và mật khẩu một lần.

- `/otp`
- `/passkey`
- `/sign-in`
- `/sign-up`
- `/sso`

### Invite auth lane

Lời mời người dùng và chấp nhận lời mời.

- `/invite`
- `/invite/accept`
- `/invite/consumed`
- `/invite/email-mismatch`
- `/invite/expired`
- `/invite/invalid`

### Recover auth lane

Quên mật khẩu và đặt lại mật khẩu.

- `/forgot-password`
- `/forgot-password/success`
- `/reset-password`
- `/reset-password/success`

### Security auth lane

Xác thực đa yếu tố, hết phiên và từ chối truy cập.

- `/access-denied`
- `/error`
- `/mfa`
- `/mfa/recovery`
- `/oauth/error`
- `/passkey/error`
- `/security/review`
- `/session-expired`
- `/sso/error`

### Verify auth lane

Xác minh email trước khi mở workspace.

- `/verify-email`
- `/verify-email/expired`
- `/verify-email/sent`
- `/verify-email/success`

### Workspace auth lane

Chọn workspace hoặc tổ chức sau xác thực.

- `/organization/select`
- `/workspace/select`
