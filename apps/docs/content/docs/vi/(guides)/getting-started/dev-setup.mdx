## Dev servers

| Surface | Command | URL |
| ------- | ------- | --- |
| Documentation | `pnpm --filter @afenda/docs dev` | http://localhost:3001/docs |
| ERP | `pnpm --filter @afenda/erp dev` | http://localhost:3000 |

Docs work is isolated on port **3001** — no ERP proxy, CSP, or auth pipeline required.

## Environment files

ERP and platform packages may require `.env.local` files. Follow package README and PAS slice handoffs for the slice you implement. **Do not commit secrets.**

For docs-only changes, no environment variables are required beyond what Next.js provides locally.

## Quality gates (pre-PR)

Run the narrowest gate for your change, then the full docs bundle before merge:

```bash
# Docs slice
pnpm --filter @afenda/docs typecheck
pnpm --filter @afenda/docs test:run
pnpm --filter @afenda/docs build
pnpm quality:boundaries
pnpm exec biome ci apps/docs
pnpm check:documentation-drift
```

Root `pnpm quality:docs` runs the docs production build in CI (docs app production build gate).

## Coding session contract

Foundation and application slices use the **afenda-coding-session** contract:

1. State Phase 0 (objective, layer, files, prohibited, authority, gates) before edits.
2. Implement one PAS slice handoff slice per session.
3. Post §11 Completion Report with gate evidence.

See [Contributing](/docs/contributing) for PAS slice handoff workflow detail.
