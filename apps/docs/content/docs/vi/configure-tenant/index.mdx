## Overview

**Tenant administrators** manage who can access the ERP and what they can do. Afenda separates **identity** (users, invites, auth routes) from **authorization** (roles, permission keys, system-admin sections).

Use the guides below in order when onboarding a new team: invite users, assign memberships, then attach the minimum role keys required for each job function.

## Principles

- Prefer **read-only keys** (`*_read`) until a workflow requires manage, post, or approve actions.
- Remove unused keys from roles to reduce privilege surface — the permission catalog below is the authority.
- Never share secrets in tickets or docs; auth configuration for operators lives under **Operate Tenant**.
