## Overview

Afenda ERP groups authentication URLs into **lanes** — access, security, verify, recover, invite, and workspace. Use this map when you need the exact path for sign-in help or when pairing with the [Sign in guide](/docs/use-erp/sign-in).

<Callout type="info" title="Tenant URL">

Always use your organization's tenant host. Paths below are suffixes such as `/sign-in` on that host.

</Callout>

## Lanes

### Access auth lane

เข้าสู่ระบบ ลงทะเบียน และรหัสผ่านครั้งเดียว

- `/otp`
- `/sign-in`
- `/sign-up`

### Invite auth lane

คำเชิญผู้ใช้และการยอมรับคำเชิญ

- `/invite`
- `/invite/accept`
- `/invite/expired`

### Recover auth lane

ลืมรหัสผ่านและรีเซ็ตรหัสผ่าน

- `/forgot-password`
- `/reset-password`
- `/reset-password/success`

### Security auth lane

การยืนยันตัวตนหลายปัจจัย หมดเซสชัน และการปฏิเสธการเข้าถึง

- `/access-denied`
- `/error`
- `/mfa`
- `/mfa/recovery`
- `/security/review`
- `/session-expired`

### Verify auth lane

ยืนยันอีเมลก่อนเปิด workspace

- `/verify-email`
- `/verify-email/expired`
- `/verify-email/sent`
- `/verify-email/success`

### Workspace auth lane

เลือก workspace หรือองค์กรหลังการยืนยันตัวตน

- `/organization/select`
- `/workspace/select`
