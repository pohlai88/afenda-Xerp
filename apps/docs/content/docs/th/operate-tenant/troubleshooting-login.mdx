## Overview

Use this runbook when users cannot sign in, complete MFA, or reach their workspace. Work top to bottom — most incidents are credential, clock skew, or membership scope issues before platform misconfiguration.

<Callout type="info" title="Gather before changing config">

Ask for the exact URL, HTTP status, time of failure, and whether the user recently changed devices or roles. Compare the URL path to the auth route catalog below before editing environment variables.

</Callout>

## Common checks

1. **Credentials** — confirm the user account is active and email is verified (`/verify-email` lane).
2. **MFA** — ensure device time is correct; use `/mfa/recovery` when authenticator access is lost.
3. **Session** — expired sessions redirect to `/session-expired`; clear cookies and retry sign-in.
4. **Access** — `/access-denied` indicates RBAC or tenant scope mismatch; review membership and roles with a tenant admin.
5. **Workspace** — users with multiple memberships must complete `/workspace/select`.
6. **SSO** — confirm IdP callback URLs and client IDs match deployment secrets (see [Environment and authentication](./environment-and-auth)).

## Escalation

If configuration looks correct but auth still fails, capture correlation IDs from observability tooling and review System Admin audit events before changing environment variables. Roll back one change at a time and re-test the full sign-in → MFA → workspace path.
