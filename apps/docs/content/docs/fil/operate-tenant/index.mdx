## Overview

**Tenant DevOps** operators configure authentication, platform connectivity, and environment variables for a deployed tenant. These guides assume you can change deployment secrets and validate auth routes — they do not replace end-user sign-in help in **Use ERP**.

Work through environment configuration first, then use the troubleshooting runbook when users report login or session failures.

## Safety

- Store secrets in `.env.secret` or your deployment vault — never in MDX, tickets, or git.
- After auth changes, validate `/sign-in`, `/mfa`, and `/invite/accept` before announcing completion.
- Capture correlation IDs and audit events before rolling back configuration.
