## Overview

Afenda ERP groups authentication URLs into **lanes** — access, security, verify, recover, invite, and workspace. Use this map when you need the exact path for sign-in help or when pairing with the [Sign in guide](/docs/use-erp/sign-in).

<Callout type="info" title="Tenant URL">

Always use your organization's tenant host. Paths below are suffixes such as `/sign-in` on that host.

</Callout>

## Lanes

### Access auth lane

Sign-in, sign-up, at one-time password entry.

- `/otp`
- `/passkey`
- `/sign-in`
- `/sign-up`
- `/sso`

### Invite auth lane

Mga imbitasyon ng user at pagtanggap ng imbitasyon.

- `/invite`
- `/invite/accept`
- `/invite/consumed`
- `/invite/email-mismatch`
- `/invite/expired`
- `/invite/invalid`

### Recover auth lane

Forgot-password at reset-password flows.

- `/forgot-password`
- `/forgot-password/success`
- `/reset-password`
- `/reset-password/success`

### Security auth lane

Multi-factor authentication, session expiry, at access-denied handling.

- `/access-denied`
- `/error`
- `/mfa`
- `/mfa/recovery`
- `/oauth/error`
- `/passkey/error`
- `/security/review`
- `/session-expired`
- `/sso/error`

### Verify auth lane

Email verification bago magbukas ang workspace.

- `/verify-email`
- `/verify-email/expired`
- `/verify-email/sent`
- `/verify-email/success`

### Workspace auth lane

Pagpili ng workspace o organization pagkatapos ng authentication.

- `/organization/select`
- `/workspace/select`
