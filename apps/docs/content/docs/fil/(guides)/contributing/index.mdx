Contributions follow **PAS slice handoffs**, **architecture authority**, and automated drift guards. This page describes the workflow — delivery specs live under `docs/PAS/<DOMAIN-FOLDER>/SLICE/` (e.g. `docs/PAS/KERNEL/SLICE/`).

## Before you code

1. Read [pas-status-index.md](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/PAS/pas-status-index.md) for canonical PAS slice status.
2. Read [afenda-runtime-truth-matrix.md](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/architecture/afenda-runtime-truth-matrix.md) for runtime evidence.
3. Copy **one** handoff block from the target PAS slice — implement one slice per session.

## afenda-coding-session contract

Every coding turn:

**Phase 0 (before edits):**

1. Objective — one sentence
2. Allowed layer — single package path
3. Files to change — explicit list
4. Prohibited — off-limits packages
5. Authority — ADR / PAS / skill
6. Acceptance gates — `pnpm` commands

**Phase 2 (before ending):** Post §11 Completion Report with drift-prevention table and gate output.

## Workflow details

<Accordions type="single">
<Accordion title="Documentation surfaces" id="documentation-surfaces">

| Surface | Path | Update when |
| ------- | ---- | ----------- |
| Governance | `docs/` at repo root | Architecture / delivery PRs |
| Published docs | `apps/docs/content/docs/` | docs app content slices |

**ADR-0012 rule:** Do not copy ADRs, registries, or PAS specs verbatim into Fumadocs MDX. Cross-link instead.

```bash
pnpm check:documentation-drift
```

</Accordion>
<Accordion title="Docs app contributions" id="docs-app-contributions">

Allowed in `apps/docs/`:

- MDX guides, onboarding, how-tos
- Docs-owned editorial palette (`docs-editorial-palette.css`)
- Fumadocs layout and source loader

Prohibited:

- `@afenda/erp` or spine package imports
- `@afenda/ui` without UI governance approval
- Brand accent in shell chrome (sidebar, search, TOC)

</Accordion>
</Accordions>

## Pull request gates

| Change type | Minimum gates |
| ----------- | --------------- |
| Docs content / TS | `pnpm --filter @afenda/docs typecheck`, `test:run`, `build` |
| Any TS/TSX | `pnpm exec biome ci apps/docs` |
| Cross-package | `pnpm quality:boundaries` |
| Status / evidence | `pnpm check:documentation-drift` |

<Callout type="note">
  Foundation PAS slices (execution spine, entitlements, system admin) run on the ERP track. Docs app work is parallel and must not block Accounting Core gate accounting gate progress.
</Callout>
