## Overview

**Integrators** connect external systems and automation to Afenda through governed **internal REST APIs** and the **module entitlement manifest**. Operation pages and module stubs are generated from repo contracts — edit `API_CONTRACTS` or catalog JSON, not the synced MDX bodies marked `DO NOT EDIT`.

## Integration principles

- Call **internal v1** routes with the same auth/session context as the ERP UI unless documented otherwise.
- Permission keys in module stubs are enforced at runtime — do not bypass with ad-hoc service accounts.
- Prefer idempotent POST patterns where the API contract exposes idempotency keys.
- For public or partner-facing APIs, wait for explicit OpenAPI publication — internal v1 is not a public surface.
