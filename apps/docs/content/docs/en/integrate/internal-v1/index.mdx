## Overview

Registry-driven **OpenAPI 3.1** catalog for governed `/api/internal/v1/**` routes. Operation pages are generated from `API_CONTRACTS` at build time — **do not edit operation MDX by hand**. To add or change an endpoint, update the contract registry and regenerate.

<Callout type="info" title="Authority">

Request shapes, auth requirements, and error envelopes follow `@afenda/erp` server API policy. This site is a rendered view of that registry, not a parallel spec.

</Callout>

## How to use this catalog

1. Find the operation card below or use search (`Ctrl+K`).
2. Open the operation page for method, path, parameters, and example envelopes.
3. Cross-check permission keys with [ERP modules](../modules) when the route touches module data.
