Contributions follow **PAS slice handoffs**, **architecture authority**, and automated drift guards. This page describes the workflow — delivery specs live under `docs/PAS/<DOMAIN-FOLDER>/SLICE/` (e.g. `docs/PAS/KERNEL/SLICE/`).

## Before you code

<Steps>

<Step>

### Read PAS slice status

Read [pas-status-index.md](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/PAS/pas-status-index.md) for canonical PAS slice closure status.

</Step>

<Step>

### Read runtime evidence

Read [afenda-runtime-truth-matrix.md](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/PAS/pas-status-index.md) for runtime evidence.

</Step>

<Step>

### Copy one handoff block

Copy **one** handoff block from the target slice under `docs/PAS/<DOMAIN-FOLDER>/SLICE/` — implement one slice per session.

</Step>

</Steps>

## afenda-coding-session contract

Every coding turn:

**Phase 0 (before edits):**

<Steps>

<Step>

### Objective

State the objective in one sentence.

</Step>

<Step>

### Allowed layer

Name the single package path you may edit.

</Step>

<Step>

### Files to change

List the explicit files expected to change.

</Step>

<Step>

### Prohibited paths

List packages and paths that are off limits.

</Step>

<Step>

### Authority

Cite the ADR, PAS standard, or skill that governs the work.

</Step>

<Step>

### Acceptance gates

List the `pnpm` commands that must pass before completion.

</Step>

</Steps>

**Phase 2 (before ending):** Post §11 Completion Report with drift-prevention table and gate output.

## Workflow details

<Accordions type="single">
<Accordion title="Documentation surfaces" id="documentation-surfaces">

| Surface | Path | Update when |
| ------- | ---- | ----------- |
| Governance | `docs/` at repo root | Architecture / PAS slice PRs |
| Published docs | `apps/docs/content/docs/` | Docs app content slices |

**ADR-0012 rule:** Do not copy ADRs, registries, or PAS specs verbatim into Fumadocs MDX. Cross-link instead.

```bash
pnpm check:documentation-drift
```

</Accordion>
<Accordion title="Docs app contributions" id="docs-app-contributions">

Allowed in `apps/docs/`:

- MDX guides, onboarding, how-tos
- Docs-owned editorial palette (`docs-editorial-palette.css`)
- Fumadocs layout and source loader

Prohibited:

- `@afenda/erp` or spine package imports
- `@afenda/ui` without UI governance approval
- Brand accent in shell chrome (sidebar, search, TOC)

</Accordion>
</Accordions>

## Pull request gates

| Change type | Minimum gates |
| ----------- | --------------- |
| Docs content / TS | `pnpm --filter @afenda/docs typecheck`, `test:run`, `build` |
| Any TS/TSX | `pnpm exec biome ci apps/docs` |
| Cross-package | `pnpm quality:boundaries` |
| Status / evidence | `pnpm check:documentation-drift` |

<Callout type="note">
  Foundation PAS slices (execution spine, entitlements, system admin) run on the ERP track. Docs app work is parallel and must not block Accounting Core gate progress (ADR-0010).
</Callout>
