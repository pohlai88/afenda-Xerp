<Steps>

<Step>

### Install at monorepo root

```bash
git clone https://github.com/pohlai88/afenda-Xerp.git
cd afenda-Xerp
pnpm install
```

</Step>

<Step>

### Configure environment

Copy env templates for ERP and database per dev-setup. Local Postgres (or Supabase) is required for data paths.

</Step>

<Step>

### Start the ERP dev server

```bash
pnpm --filter @afenda/erp dev
```

Default port **3000**.

</Step>

<Step>

### Run quality gates before PR

```bash
pnpm --filter @afenda/erp typecheck
pnpm --filter @afenda/erp test:run
pnpm quality:boundaries
pnpm check:documentation-drift
```

</Step>

</Steps>

## Layer discipline

When changing ERP UI:

1. Edit `apps/erp/` first — not shared packages unless required
2. Ask before editing `packages/ui` primitives (Governed UI)
3. One bounded slice per session — follow [ARCH index](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/PAS/arch-status-index.md)
