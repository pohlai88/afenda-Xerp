## Purpose

Afenda **Presentation Lab** for `@afenda/shadcn-studio` — colocated primitives, MCP blocks, theme presets, and flat-block fixtures. Storybook consumes studio sources; it does **not** own ERP runtime.

**Retired for ERP:** `@afenda/ui`, `@afenda/appshell`, `@afenda/metadata-ui`, PAS-005 CSS gates.

## Run locally

```bash
pnpm storybook
```

Or: `pnpm --filter @afenda/storybook storybook` (port **6006**, exact-port).

## Story layers

| Layer | Path | Role |
| --- | --- | --- |
| Primitives | `components/ui/*.stories.tsx` | Colocated CSF3 + `play` smoke |
| Curated blocks | `shadcn-studio-blocks.stories.tsx` | Dark variants, fixtures, Figma links |
| Flat blocks | `shadcn-studio-blocks-flat.stories.tsx` | 33 prop-driven MCP blocks (curated fixtures) |
| Auto blocks | `shadcn-studio-blocks-auto.stories.tsx` | Codegen — self-contained page blocks |
| Theme lab | `shadcn-studio-theme-lab.stories.tsx` | Preset matrix + ThemeCustomizer |

Regenerate auto stories after MCP install: `pnpm storybook generate`

## Boundary rules

- Storybook **consumes** `@afenda/shadcn-studio` — no ERP business logic
- ERP imports blocks from `@afenda/shadcn-studio` only (ADR-0027)
- Lab parameters live in `@afenda/shadcn-studio/lab` — not for production ERP imports

## Quality gates

```bash
pnpm storybook:generate
pnpm check:storybook-block-coverage
pnpm --filter @afenda/storybook typecheck
pnpm check:studio-import-zones
pnpm storybook:build
pnpm test:storybook:run
```

CI: `.github/workflows/storybook-lab.yml`
