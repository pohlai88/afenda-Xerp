## Overview

Afenda ERP groups authentication URLs into **lanes** — access, security, verify, recover, invite, and workspace. Use this map when you need the exact path for sign-in help or when pairing with the [Sign in guide](/docs/use-erp/sign-in).

<Callout type="info" title="Tenant URL">

Always use your organization's tenant host. Paths below are suffixes such as `/sign-in` on that host.

</Callout>

## Lanes

### Access auth lane

登录、注册与一次性密码入口。

- `/otp`
- `/sign-in`
- `/sign-up`

### Invite auth lane

用户邀请与接受邀请。

- `/invite`
- `/invite/accept`
- `/invite/expired`

### Recover auth lane

忘记密码与重置密码流程。

- `/forgot-password`
- `/reset-password`
- `/reset-password/success`

### Security auth lane

多因素认证、会话过期与访问拒绝页面。

- `/access-denied`
- `/error`
- `/mfa`
- `/mfa/recovery`
- `/security/review`
- `/session-expired`

### Verify auth lane

在进入工作区前完成邮箱验证。

- `/verify-email`
- `/verify-email/expired`
- `/verify-email/sent`
- `/verify-email/success`

### Workspace auth lane

认证后选择工作区或组织。

- `/organization/select`
- `/workspace/select`
