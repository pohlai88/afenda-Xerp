贡献遵循 **交付 TIP 切片**、**架构权威** 与自动化漂移守卫。本页说明工作流 — 交付规格仍以 `docs/PAS/slice/` 为准。

## 编码前

1. 阅读 [tip-status-index.md](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/PAS/README.md) 了解 TIP 状态。
2. 阅读 [afenda-runtime-truth-matrix.md](https://github.com/pohlai88/afenda-Xerp/blob/main/docs/architecture/afenda-runtime-truth-matrix.md) 了解运行时证据。
3. 从目标 TIP 复制 **一个** 交接块 — 每个会话只实现一个切片。

## afenda-coding-session 契约

每次编码回合：

**Phase 0（编辑前）：**

1. Objective — 一句话目标
2. Allowed layer — 单一包路径
3. Files to change — 明确文件列表
4. Prohibited — 禁止触及的包
5. Authority — ADR / TIP / skill
6. Acceptance gates — `pnpm` 命令

**Phase 2（结束前）：** 发布 §11 完成报告，含漂移防护表与 gate 输出。

## Pull request 门禁

| 变更类型 | 最低 gate |
| --- | --- |
| 文档内容 / TS | `pnpm --filter @afenda/docs typecheck`、`test:run`、`build` |
| 任意 TS/TSX | `pnpm exec biome ci apps/docs` |
| 跨包 | `pnpm quality:boundaries` |
| 状态 / 证据 | `pnpm check:documentation-drift` |

<Callout type="note">
  基础 TIP（执行 spine、entitlements、system admin）走 ERP 轨道。TIP-032 文档工作并行进行，不得阻塞 TIP-014+ 会计 gate 进度。
</Callout>
