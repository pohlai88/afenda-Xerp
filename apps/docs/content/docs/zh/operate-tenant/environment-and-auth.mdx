## Overview

Tenant DevOps operators configure authentication and platform connectivity through **governed environment variables**. Names and descriptions are synced to this site; **values are never exported** to documentation or git.

<Callout type="warn" title="Secrets">

Never commit secret values to documentation or source control. Use `.env.secret`, your deployment vault, and `pnpm env:sync` on developer machines. Public keys and URLs belong in `.env.config` only when the repo documents them as non-secret.

</Callout>

## Auth-related variables

Review Supabase public keys, Better Auth URL, and OAuth client IDs in `.env.config`. Pair secrets (signing keys, database passwords) from `.env.secret` only on deployment targets.

After any auth-related change:

1. Redeploy or restart the ERP process so env is re-read.
2. Walk `/sign-in` → `/mfa` (if enabled) → `/workspace/select` with a test user.
3. Compare observed routes to the catalog below.
