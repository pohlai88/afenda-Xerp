The Afenda monorepo uses **pnpm workspaces** under `apps/` (applications) and `packages/` (shared libraries). The canonical package registry lives at `docs/architecture/package-registry.md` — this page summarizes engineer navigation; do not treat it as a registry substitute.

## Applications

| Package | Path | Port | Role |
| ------- | ---- | ---- | ---- |
| `@afenda/erp` | `apps/erp` | 3000 | Primary ERP delivery surface |
| `@afenda/docs` | `apps/docs` | 3001 | Fumadocs implementation documentation (this site) |
| `@afenda/storybook` | `apps/storybook` | — | Design-system and shell Storybook |

**Boundary:** `@afenda/erp` must not depend on `@afenda/docs`. The docs app is a leaf application.

## Platform spine (selected)

| Package | Path | Authority |
| ------- | ---- | --------- |
| `@afenda/database` | `packages/database` | Platform — schema and persistence |
| `@afenda/kernel` | `packages/kernel` | Platform — IDs, Result, execution contracts |
| `@afenda/permissions` | `packages/permissions` | Platform — authorization |
| `@afenda/execution` | `packages/execution` | Platform — durable execution / outbox |
| `@afenda/entitlements` | `packages/entitlements` | Integration — feature manifest |

## Design and UI

| Package | Path | Notes |
| ------- | ---- | ----- |
| `@afenda/ui` | `packages/ui` | Governed UI primitives (TIP-004) |
| `@afenda/appshell` | `packages/appshell` | ERP shell chrome |
| `@afenda/design-system` | `packages/design-system` | Tokens — **not** imported by `@afenda/docs` shell |

## Layer discipline

When changing UI or docs:

1. **App wiring first** — `apps/erp/` or `apps/docs/` before shared packages.
2. **Ask before editing** `packages/ui` primitives — governed by TIP-004.
3. **One slice per session** — database → platform → app dependency order for foundation TIPs.

## Dependency enforcement

```bash
pnpm quality:boundaries
```

Unapproved `@afenda/*` edges fail CI. `@afenda/docs` currently has **zero** approved runtime workspace dependencies (dev-only `@afenda/typescript-config`).
