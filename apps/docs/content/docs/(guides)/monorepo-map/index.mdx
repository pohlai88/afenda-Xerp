## Overview

The Afenda monorepo uses **pnpm workspaces** under `apps/` (applications) and `packages/` (shared libraries). The canonical package registry lives at `docs/architecture/package-registry.md` — this page summarizes engineer navigation; do not treat it as a registry substitute.

## Layer discipline

When changing UI or docs:

1. **App wiring first** — `apps/erp/` or `apps/docs/` before shared packages.
2. **Contract reference** — see [Docs app contracts](/docs/monorepo-map/docs-contracts) for `@afenda/docs` TypeScript surfaces.
3. **Ask before editing** `packages/ui` primitives — governed by TIP-004.
4. **One slice per session** — database → platform → app dependency order for foundation TIPs.
5. **Governance docs** — ADRs and registries live under repo-root `docs/`; link, do not duplicate.

## Dependency enforcement

```bash
pnpm quality:boundaries
```

Unapproved `@afenda/*` edges fail CI. `@afenda/docs` currently has **zero** approved runtime workspace dependencies (dev-only `@afenda/typescript-config`).
