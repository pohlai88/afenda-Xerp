# Troubleshooting login

Use this runbook when users cannot sign in, complete MFA, or reach their workspace.

## Common checks

1. **Credentials** — confirm the user account is active and email is verified (`/verify-email` lane).
2. **MFA** — ensure device time is correct; use `/mfa/recovery` when authenticator access is lost.
3. **Session** — expired sessions redirect to `/session-expired`; clear cookies and retry sign-in.
4. **Access** — `/access-denied` indicates RBAC or tenant scope mismatch; review membership and roles.
5. **Workspace** — users with multiple memberships must complete `/workspace/select`.
