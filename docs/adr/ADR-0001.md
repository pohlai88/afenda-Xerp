# ADR-0001 — Phase 1 Foundation Redefinition

| Field | Value |
|-------|-------|
| **Status** | Accepted |
| **Date** | 2026-06-20 |
| **Owner** | Architecture Authority |
| **Supersedes** | Master Plan Phase 1 TIP Structure; TIP-001 through TIP-012 original definitions |
| **Does Not Supersede** | TIP numbering; TIP-013 Accounting Core Contracts; Phase 2, Phase 3, Phase 4 |

---

## Context

Implementation review of the Platform Foundation revealed that the original Phase 1 roadmap was primarily oriented around technical infrastructure.

While technically valid, the sequence allowed implementation to begin before the following were fully defined:

- Ownership and authority
- Design governance
- Metadata governance
- AppShell governance
- ERP operating governance

This created architectural drift risk across:

- Design System
- Metadata UI
- AppShell
- Permissions
- Domain Ownership
- Package Ownership
- AI-assisted Development

**Objective:** Redefine Phase 1 so that all governance, ownership, platform authority, and operating spine concerns are completed before Accounting Core begins.

Accounting is not permitted to become the first domain.  
The ERP operating platform must become the first domain.

---

## Decision

Phase 1 shall be redefined.

TIP identifiers remain unchanged. TIP-001 through TIP-012 are redefined as follows. TIP-013 Accounting Core remains the first business-domain TIP.

---

## Phase 1 TIP Redefinition

### TIP-001 — Architecture Authority

**Purpose:** Establish platform truth.

**Deliverables**
- Architecture Map
- Ownership Map
- Dependency Map
- ADR Framework
- Package Boundaries
- Layer Boundaries

**Must Define**
- Platform ownership
- Package ownership
- Dependency direction
- Architectural layers
- Prohibited dependencies

**Acceptance Gate**  
No package may exist without ownership.  
No dependency may exist without approval.

---

### TIP-002 — AI Development Governance

**Purpose:** Establish AI-assisted development rules.

**Deliverables**
- AI Governance Contract
- Anti-Drift Contract
- Implementation Policy
- Code Generation Policy

**Must Define**
- Allowed AI patterns
- Forbidden AI patterns
- Generation boundaries
- Implementation standards

**Acceptance Gate**  
AI may not invent architecture.  
AI may not invent ownership.  
AI may not invent contracts.

---

### TIP-003 — Design System Authority

**Purpose:** Establish visual ownership.

**Deliverables**
- Design System Authority Contract

**Must Define**
- Token ownership
- Variant ownership
- Recipe ownership
- Component ownership
- Slot ownership

**Acceptance Gate**  
Visual authority is fully defined.

---

**Purpose:** Establish anti-drift UI contracts.

**Deliverables**

### TIP-011 — Execution Foundation

**Purpose:** Build platform execution services.

**Deliverables**

- Drizzle
- Storage
- Trigger.dev
- Outbox
- Observability
- Audit Infrastructure

**Acceptance Gate**  
Platform execution services are operational.

---

### TIP-012 — ERP Operating Spine

**Purpose:** Create mandatory runtime architecture used by every ERP domain.

**Deliverables**
- Execution Context
- Permission Context
- Audit Context
- Approval Context
- Metadata Context
- Feature Flag Context
- Entitlement Context
- Correlation Context

**Must Define**

Protected Action Lifecycle:

```
Validation
  → Authorization
  → Policy
  → Execution
  → Audit
  → Observability
  → Event Publication
```

**Acceptance Gate**  
Every future ERP module must execute through the operating spine.  
No business domain may bypass the operating spine.

---

## Phase 1 Exit Gate

TIP-013 Accounting Core may not begin until:

| Condition | Required |
|-----------|----------|
| TIP-001 through TIP-012 complete | ✅ |
| Ownership maps approved | ✅ |
| Design governance approved | ✅ |
| Metadata governance approved | ✅ |
| AppShell governance approved | ✅ |
| Platform governance approved | ✅ |
| Master data governance approved | ✅ |
| Identity and authorization operational | ✅ |
| Execution services operational | ✅ |
| ERP Operating Spine operational | ✅ |

---

## Consequence

Phase 1 no longer represents infrastructure setup.

Phase 1 represents **complete platform governance and operating foundation**.

Accounting becomes the first ERP business domain built on top of a governed platform — rather than becoming part of the foundation itself.
