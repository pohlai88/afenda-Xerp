# Slice B14 — Localization and Global Format Vocabulary (PAS-001 §4.5)

**Prerequisite:** Slice B4 — primitive references (`docs/PAS/slice/b4.md`, `Status: Delivered`)

**Status:** Delivered

**Type:** Implementation

**Risk class:** Low

**Clean Core impact:** A→A (wire trust boundary + timezone hardening on existing B4 primitives)

## Purpose

Complete PAS §4.5 localization vocabulary runtime:

1. Harden `TimezoneId` parser to reject UTC offsets (`+07:00`, `-05:00`) per PAS §4.5.
2. Add `WireLocalizationContext`, `parseLocalizationContext`, and `serializeLocalizationContext` on the kernel trust boundary.
3. Add dedicated localization context contract tests.
4. Export wire helpers from `@afenda/kernel` public surface.

## Frozen rules (PAS §4.5)

1. Kernel owns the vocabulary and context shape.
2. Kernel does not decide the selected locale, timezone, date format, or number format.
3. Kernel does not format dates or numbers at runtime.
4. Kernel does not parse user-entered dates.
5. Kernel does not localize UI labels.
6. Kernel does not own translation files.
7. `TimezoneId` values must follow IANA tzdata identifier format; UTC offsets are not valid `TimezoneId` values.
