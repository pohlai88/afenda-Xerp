## Rules frozen

1. `FiscalCalendarId` and `FiscalPeriodId` must not be added to `ID_FAMILIES` unless a future ADR explicitly promotes them.
2. Forbidden symbols live in `FORBIDDEN_PLATFORM_FLOOR_ID_SYMBOLS` — not a separate policy registry file.
3. Both fiscal IDs may exist only on `@afenda/kernel/accounting-domain` as domain vocabulary — not platform-floor canonical enterprise IDs.
4. Forbidden fiscal IDs must not receive `<prefix>_<ulid_body>` parsers, prefixes, or rows in `ID_FAMILIES`.
5. Main `@afenda/kernel` barrel and `identity/families/*` must not export `export type FiscalCalendarId` or `export type FiscalPeriodId`.
6. Gate `pnpm check:forbidden-platform-ids` enforces platform-floor export prohibition.
