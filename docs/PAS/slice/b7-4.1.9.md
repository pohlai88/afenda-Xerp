# Recommended PAS-001 §4.1.8 Drop-in

```md id="u5gjke"
## 4.1.8 Validation rules

Family `parse*` functions enforce §4.1.3 Canonical format at trust boundaries.

A parser must reject:

| Invalid condition     | Example                                                    | Expected result |
| --------------------- | ---------------------------------------------------------- | --------------- |
| Wrong family prefix   | `cus_01ARZ3NDEKTSV4RRFFQ69G5FAV` passed to `parseTenantId` | Reject          |
| Invalid body length   | `ten_01ARZ3NDEKTSV4RRFFQ69G5FA`                            | Reject          |
| Invalid body charset  | `ten_01ARZ3NDEKTSV4RRFFQ69G5FAI`                           | Reject          |
| Missing separator     | `ten01ARZ3NDEKTSV4RRFFQ69G5FAV`                            | Reject          |
| Uppercase prefix      | `TEN_01ARZ3NDEKTSV4RRFFQ69G5FAV`                           | Reject          |
| Unknown prefix        | `abc_01ARZ3NDEKTSV4RRFFQ69G5FAV`                           | Reject          |
| Empty value           | `""`                                                       | Reject          |
| Whitespace-only value | `"   "`                                                    | Reject          |

Rules:

1. Family parsers must trim outer whitespace before validation.
2. Family parsers must reject empty or whitespace-only values.
3. Family parsers must reject IDs whose prefix does not match the requested family.
4. Family parsers must reject unknown prefixes.
5. Family parsers must reject invalid ULID body length or charset.
6. The only allowed cast from string to branded canonical ID is inside Kernel parser internals after validation.
```

Recommended helper:

```ts id="welhot"
export function parseCanonicalId<TFamily extends IdFamily>(
  value: string,
  family: TFamily
): CanonicalEnterpriseId<TFamily> {
  const raw = value.trim();

  if (!raw) {
    throw new InvalidCanonicalIdError(
      `${ID_FAMILIES[family].typeName} is required.`
    );
  }

  if (!CANONICAL_ID_PATTERN.test(raw)) {
    throw new InvalidCanonicalIdError(
      `${ID_FAMILIES[family].typeName} has invalid canonical ID format.`
    );
  }

  const [actualPrefix] = raw.split("_");
  const registeredPrefixes = new Set(
    Object.values(ID_FAMILIES).map((entry) => entry.prefix)
  );

  if (!registeredPrefixes.has(actualPrefix as IdPrefix)) {
    throw new InvalidCanonicalIdError(
      `${ID_FAMILIES[family].typeName} uses an unknown canonical ID prefix.`
    );
  }

  const expectedPrefix = ID_FAMILIES[family].prefix;

  if (actualPrefix !== expectedPrefix) {
    throw new InvalidCanonicalIdError(
      `${ID_FAMILIES[family].typeName} must start with ${expectedPrefix}_.`
    );
  }

  return raw as CanonicalEnterpriseId<TFamily>;
}
```
