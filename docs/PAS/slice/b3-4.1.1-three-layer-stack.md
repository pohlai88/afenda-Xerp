## Rules frozen

1. Exactly three layers: `internal-pk`, `canonical-enterprise-id`, `tenant-human-reference`.
2. Internal PK owner is `@afenda/database`; ingress via `parseInternalEntityPk`.
3. Canonical enterprise ID owner is `@afenda/kernel` + database CHECK; ingress via family `parse*`.
4. Tenant human reference owner is Domain / System Admin; ingress via scope `parse*` helpers — kernel classifies only.
5. Layers must not cross-contaminate at trust boundaries (see `prohibitedCrossLayerPatterns`).
