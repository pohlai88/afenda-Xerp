## Rules frozen

1. Better Auth `auth_user.id` is `AuthSubjectId` ŌĆö OAuth/login identity with text PK; never a `usr_*` enterprise ID.
2. ERP actor `UserId` is the Kernel `usr_*` canonical enterprise ID ŌĆö distinct from `AuthSubjectId`.
3. Platform `users.id` internal uuid uses `InternalEntityPk` at trust boundaries.
4. Auth-subject parse rejects canonical enterprise IDs and tenant human references.
5. Auth actor bridge fields must not conflate `authSubjectId`, `userPk`, and `userId`.
6. Wire JSON remains plain strings; trusted shapes use `parseAuthSubjectId`, `parseInternalEntityPk`, and `parseUserId` at ingress.
7. `AuthSubjectId` must not enter `ID_FAMILIES` or enterprise ID registries.
