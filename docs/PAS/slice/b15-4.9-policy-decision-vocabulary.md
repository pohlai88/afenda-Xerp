# Slice B15 — Policy Decision Vocabulary (PAS-001 §4.9)

**Prerequisite:** Slice B14 — Localization vocabulary (`docs/PAS/slice/b14-4.5-localization.md`, `Status: Delivered`)

**Status:** Delivered

**Type:** Implementation

**Risk class:** Low

**Clean Core impact:** A→A (vocabulary registry + wire-safe discriminated union on existing bare unions)

## Purpose

Codify PAS §4.9 policy decision vocabulary as typed kernel registry data:

1. Const arrays with derived types for `PolicyDecisionKind` (4 literals) and `PolicyDenialReason` (9 literals).
2. Type guards, non-throwing narrow helpers, and authority/ownership metadata.
3. Wire-safe `PolicyDecision` discriminated union with structural `isPolicyDecision` guard.
4. Comprehensive contract tests and public `@afenda/kernel` exports.

## Frozen rules (PAS §4.9)

1. Kernel owns policy decision vocabulary.
2. `@afenda/permissions` owns policy evaluation.
3. `@afenda/entitlements` owns commercial plan and capability evaluation.
4. API governance owns HTTP mapping.
5. ERP owns route/action enforcement.
6. Database owns role/permission persistence.
7. Kernel must not implement evaluation, entitlements, HTTP status mapping, ERP enforcement, or DB persistence.
