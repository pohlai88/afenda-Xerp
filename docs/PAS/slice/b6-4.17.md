# PAS-001 §4.1.7 Drop-in

````md
## 4.1.7 Trust boundaries

Wire JSON payloads must keep IDs as plain strings.

Canonical enterprise IDs become trusted internal values only after passing through Kernel `parse*` functions.

Approved:

```ts
const tenantId = parseTenantId(params.tenantId);
const customerId = parseCustomerId(body.customerId);
````

Prohibited in `apps/*`, domain packages, and feature packages:

```ts
const tenantId = wire as TenantId;
const tenantId = customerId as CustomerId;
const tenantId = brandTenantId(untrusted);
```

Rules:

1. API, event, import, route, and database ingress must parse string IDs through Kernel `parse*` functions.
2. Wire payloads remain plain strings.
3. Internal application services may use branded canonical ID types after parsing.
4. Egress must serialize branded IDs back to strings.
5. New code must use `parse*` for untrusted input.
6. `brand*` / `to*` wrappers are transitional compatibility only.
7. If `brand*` / `to*` remain, they must call `parse*` internally and must not cast directly.
8. Raw `as TenantId`, `as CustomerId`, or `as CanonicalEnterpriseId<any>` is prohibited outside Kernel parser internals.


```ts
import { unbrand } from "../brand";
import type { CanonicalEnterpriseId } from "../canonical";
import type { IdFamily } from "../registry";
import { parseCanonicalId } from "../canonical";

export type WireCanonicalId = string;

export function parseWireCanonicalId<TFamily extends IdFamily>(
  value: WireCanonicalId,
  family: TFamily
): CanonicalEnterpriseId<TFamily> {
  return parseCanonicalId(value, family);
}

export function serializeCanonicalId<TFamily extends string>(
  value: CanonicalEnterpriseId<TFamily>
): WireCanonicalId {
  return unbrand(value);
}
```

# Transitional wrapper policy

If you still keep `brand*` / `to*`, make them wrappers only:

```ts
export function toTenantId(value: string): TenantId {
  return parseTenantId(value);
}

/**
 * @deprecated Use parseTenantId for untrusted input.
 */
export function brandTenantId(value: string): TenantId {
  return parseTenantId(value);
}
```

Do **not** allow:

```ts
return value as TenantId;
```

inside wrappers.

# Required tests

Add:

```txt
packages/kernel/src/identity/wire/__tests__/identity-wire.contract.test.ts
```

Minimum tests:

```ts
describe("identity wire boundary", () => {
  it("parses valid wire tenant ID", () => {
    expect(() =>
      parseWireCanonicalId("ten_01ARZ3NDEKTSV4RRFFQ69G5FAV", "tenant")
    ).not.toThrow();
  });

  it("rejects wrong family at wire boundary", () => {
    expect(() =>
      parseWireCanonicalId("cus_01ARZ3NDEKTSV4RRFFQ69G5FAV", "tenant")
    ).toThrow();
  });

  it("rejects malformed wire ID", () => {
    expect(() => parseWireCanonicalId("tenant-001", "tenant")).toThrow();
  });

  it("serializes trusted canonical ID back to string", () => {
    const tenantId = parseWireCanonicalId(
      "ten_01ARZ3NDEKTSV4RRFFQ69G5FAV",
      "tenant"
    );

    expect(serializeCanonicalId(tenantId)).toBe(
      "ten_01ARZ3NDEKTSV4RRFFQ69G5FAV"
    );
  });
});
```
