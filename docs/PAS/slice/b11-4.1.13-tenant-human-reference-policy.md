## Rules frozen

1. Tenant human references are not canonical enterprise IDs and must not enter `ID_FAMILIES`.
2. Kernel classifies and parses human references; Kernel must not generate or allocate human numbers.
3. Human reference parsers must reject canonical enterprise ID wire shapes (`emp_*`, `cus_*`, …).
4. Database registry documents composite uniqueness for documents: `(tenant_id, document_type, document_no)`.
5. Deferred entity tables (employees, customers, suppliers, documents, assets) stay `persistenceStatus: "deferred"` until domain FDRs land schemas.
6. ADR Slice F acceptance gates alias existing platform enforcement — no duplicate scanner logic.
