## Runtime delivered

| Concern | Authority |
| --- | --- |
| Trusted identity | `AuditEntityIdentity<TFamily>` |
| Wire shape | `WireAuditEntityIdentity` |
| Ingress parse | `parseAuditEntityIdentity` |
| Egress serialize | `serializeAuditEntityIdentity` |
| Internal PK brand | `InternalEntityPk`, `parseInternalEntityPk` |
| Public exports | `@afenda/kernel` via `identity/index.ts` |

## Rules frozen

1. `entityId` / `tenantId` hold canonical enterprise IDs only.
2. `entityPk` / `tenantPk` hold internal UUID PK strings only.
3. Canonical fields parse through family `parse*` / `parseTenantId`.
4. PK fields reject canonical enterprise ID, tenant human reference, and non-UUID-v7 wire shapes.
5. Wire JSON remains plain strings; serializers use `serializeCanonicalId` for canonical fields.
