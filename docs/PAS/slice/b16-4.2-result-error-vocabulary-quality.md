## Objective

Quality pass over the PAS §4.2 Result and Error Vocabulary contracts:

1. Add RFC 9457 `status?: number` field to `ProblemDetail` (additive, backward-compatible).
2. Simplify `AppErrors.validation` and `AppErrors.internal` conditional spread expressions (zero behavior change).
3. Expand `result.contract` test coverage: type narrowing via `isOk`/`isErr`, `AppError` as error type.
4. Expand `problem-detail.contract` test coverage: full RFC 9457 shape, minimal fields, `status` field.

PAS §4.2 frozen rules are fully preserved: discriminated-union style, `AppErrors.*` factory style, `userMessage` semantics, kernel error ownership, no `AppError` replacement by `ProblemDetail`.
