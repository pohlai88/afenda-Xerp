## Rules frozen

1. Kernel owns the canonical `ExecutionContext` shape.
2. Required concepts: `executionId`, `correlationId`, `source`, `startedAt`, `actorId`, `tenantId`, `companyId`, `organizationId`.
3. Approved additive fields: `traceId`, `spanId` (plain strings, nullable).
4. `ExecutionContextSource` vocabulary: `"api" | "cron" | "event" | "manual" | "system" | "outbox" | "job"`.
5. Kernel must not import OpenTelemetry; observability enriches trace/span outside kernel.
6. `correlationId` follows a request/event/job/action chain; `executionId` identifies one execution attempt.
