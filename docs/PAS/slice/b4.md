Primitive references are globally shared reference values used across modules.

They are not canonical enterprise IDs and must not use the `<prefix>_<ulid_body>` parser.

| `CountryCode`  | `MY`, `SG`, `VN`                        | ISO 3166-1 alpha-2-style uppercase code | Kernel |

# 4. Primitive parser examples

## `currency-code.contract.ts`

```ts
import type { PrimitiveReference } from "./primitive-brand.contract";

export type CurrencyCode = PrimitiveReference<"CurrencyCode">;

const CURRENCY_CODE_PATTERN = /^[A-Z]{3}$/;

export function parseCurrencyCode(value: string): CurrencyCode {
  const raw = value.trim().toUpperCase();

  if (!CURRENCY_CODE_PATTERN.test(raw)) {
    throw new Error("CurrencyCode must be a 3-letter uppercase ISO-style code.");
  }

  return raw as CurrencyCode;
}
```

## `country-code.contract.ts`

```ts
import type { PrimitiveReference } from "./primitive-brand.contract";

export type CountryCode = PrimitiveReference<"CountryCode">;

const COUNTRY_CODE_PATTERN = /^[A-Z]{2}$/;

export function parseCountryCode(value: string): CountryCode {
  const raw = value.trim().toUpperCase();

  if (!COUNTRY_CODE_PATTERN.test(raw)) {
    throw new Error("CountryCode must be a 2-letter uppercase ISO-style code.");
  }

  return raw as CountryCode;
}
```

## `timezone-id.contract.ts`

```ts
import type { PrimitiveReference } from "./primitive-brand.contract";

export type TimezoneId = PrimitiveReference<"TimezoneId">;

export function parseTimezoneId(value: string): TimezoneId {
  const raw = value.trim();

  if (!raw) {
    throw new Error("TimezoneId is required.");
  }

  if (!raw.includes("/") && raw !== "UTC") {
    throw new Error("TimezoneId must be an IANA-style timezone ID.");
  }

  return raw as TimezoneId;
}
```

For timezone, avoid pretending regex alone is perfect. A later runtime validator can use platform-supported timezone checks.

# 5. Important ownership decision

For these primitives:

| Primitive      |               Kernel owns? |                             Domain owns? | Why                                                            |
| -------------- | -------------------------: | ---------------------------------------: | -------------------------------------------------------------- |
| `LocaleCode`   |                        Yes |                                       No | Used everywhere                                                |
| `TimezoneId`   |                        Yes |                                       No | Used everywhere                                                |
| `DateFormat`   |                        Yes |                                       No | Global presentation setting                                    |
| `NumberFormat` |                        Yes |                                       No | Global presentation setting                                    |
| `CurrencyCode` |                        Yes |                      Finance may consume | Global reference; finance does not own identity                |
| `CountryCode`  |                        Yes |                                       No | Used by HRM, CRM, tax, address, logistics                      |
| `UomCode`      | Yes for primitive contract | Inventory/domain may own allowed catalog | UOM semantics used across purchasing, inventory, manufacturing |

Important nuance:

```txt
Kernel owns the primitive type.
Domain/System Admin may own allowed tenant configuration.
```

Example:

```txt
UomCode type = Kernel
Allowed UOM catalog per tenant = System Admin / Inventory policy
```
