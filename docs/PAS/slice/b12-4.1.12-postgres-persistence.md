## Rules frozen

1. Kernel canonical enterprise IDs are **not** PostgreSQL primary keys.
2. `id uuid` is the internal PK (UUID v7 default via `uuid_generate_v7()`).
3. Foreign keys reference internal UUID primary keys only — never `enterprise_id`.
4. `enterprise_id text` stores the Kernel-governed canonical enterprise ID with family CHECK + unique lookup index.
5. API, audit, events, logs, imports, exports, and cross-package wire contracts use `enterprise_id`.
6. Joins, foreign keys, RLS policies, and internal relational integrity use UUID keys.
7. CHECK patterns in `@afenda/database` must parity-validate against kernel via `pnpm check:enterprise-id-db-parity`.
8. Production ULID body minting injects `@afenda/database` `generateUlidBody()` through ERP `CanonicalIdBodyGenerator` (ADR-0024) — kernel does not mint without injected generator.
