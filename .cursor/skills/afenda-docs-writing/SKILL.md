# Afenda Docs Writing

Governed MDX authoring for `apps/docs/content/docs/` on Fumadocs MDX + Afenda editorial typography.

## When to use

- Creating or revising MDX under `apps/docs/content/docs/`
- Reviewing docs PRs for structure, enrichment, or duplicate titles
- Agent work on reader sections (`use-erp`, `configure-tenant`, `operate-tenant`, `integrate`) or `build-afenda`

## Fumadocs content contract (non-negotiable)

1. **`title` and `description` live in frontmatter only** — they render as `DocsTitle` and `DocsDescription`.
2. **Do not repeat `title` as `# Heading` in the body.** Fumadocs treats frontmatter `title` as the page H1.
3. **Body starts at `##`** (Overview, Steps, Reference, etc.).
4. **`description` is metadata** — one quiet summary line. It is **not** a lead paragraph; do not style or write it like marketing copy.
5. **Enrich, do not collapse** — removing a duplicate H1 must be followed by an **Overview** (or task-specific H2) with useful context, not an empty page.

Official reference: [Fumadocs Markdown](https://fumadocs.dev/docs/markdown) · [Page conventions](https://fumadocs.dev/docs/headless/page-conventions)

## Afenda typography roles (UI-first — content ≤ chrome)

Four sizes only. Content must **never** read larger than nav, tabs, or sidebar.

**Palette pairing:** The luxury editorial palette uses soft ink and warm neutrals — spacing carries hierarchy because borders are quiet (18–55% opacity). Use semantic roles, not ad-hoc margins.

| Role | Token | Size | Use |
|------|-------|------|-----|
| Chrome gap | `--docs-gap-chrome` | 12px | Title · description · actions |
| Block gap | `--docs-gap-block` | 16px | Prose stack · card grid |
| Section gap | `--docs-gap-section` | 24px | h2 breaks · footer · content entry |
| Inline gap | `--docs-gap-inline` | 8px | Icon+label · callout title+body |
| Component inset | `--docs-inset-component` | 16px | Cards · tabs |
| Comfort inset | `--docs-inset-comfort` | 24px | Callouts |

Do not use tight UI spacing (8–12px block gaps) with this palette — low-contrast ink needs 16px blocks and 1.65 line-height.

| Level | Token | Size | Use |
|-------|-------|------|-----|
| Meta | `--docs-text-meta` | 12px | `description`, captions, TOC label |
| UI | `--docs-text-ui` | 13px | Body, lists, h3/h4, nav, tabs |
| Section | `--docs-text-section` | 14px | `##` headings only |
| Title | `--docs-text-title` | 16px | Page title (`DocsTitle`) — max content size |

Hierarchy inside prose uses **weight and colour**, not larger font sizes. Headings inside Fumadocs `<Tabs>` stay at UI size (13px).

| Role | Class / surface | Use |
|------|-----------------|-----|
| Display | `DocsTitle` / frontmatter `title` | Page title only (16px max) |
| Summary | `docs-type-summary` via `DocsDescription` | Frontmatter `description` (12px) |
| Deck | `docs-type-deck` | Home hero lead only (13px) |
| Body | `.prose` from `##` onward | All MDX content (13px) |

## Frontmatter patterns

### Task article (reader lane)

```yaml
---
title: Sign in to Afenda ERP
description: Sign-in paths, MFA, and workspace selection for end users.
audience: end-user
relatedRoutes:
  - /sign-in
  - /mfa
catalogBindings:
  - auth-routes
---
```

`audience`: `end-user` | `tenant-admin` | `tenant-devops` | `engineer`

### Index / hub page

```yaml
---
title: Use ERP
description: End-user guides for signing in, workspaces, and daily ERP tasks.
---
```

### Generated / synced (do not hand-edit body)

Files containing `{/* DO NOT EDIT — generated by` are owned by sync scripts. Change the generator or catalog, not the MDX.

## Page skeleton (hand-authored)

```mdx
---
title: Page Title
description: One-line metadata summary for search and page chrome.
audience: tenant-admin
---

## Overview

Two to four sentences: who this is for, what they will accomplish, and how it fits Afenda lanes (auth, system-admin, modules, etc.).

<Callout type="info" title="Optional tip">
  Actionable constraint or link — not a second title.
</Callout>

## Primary task or section

<Steps>
<Step>

### Step title

Step detail with routes in backticks (`/system-admin/users`).

</Step>
</Steps>

## Reference

<GeneratedReference catalog="permissions" />
```

## Enrichment checklist (every edit)

- [ ] Frontmatter `title` + `description` present
- [ ] No `#` line matching `title` (case-insensitive, trimmed)
- [ ] **Overview** H2 opens the body with real context
- [ ] Task pages use **Steps** or numbered procedures with ERP routes
- [ ] Reference sections use **GeneratedReference** / OpenAPI — not hand-maintained tables
- [ ] **Callout** for warnings (`warn`), scope (`info`), or blockers — sparingly
- [ ] **Cards** on index pages with descriptive `description` props (not one-word stubs)
- [ ] Links to repo-root `docs/` governance only when citing ADR/PAS — runtime docs stay in this site
- [ ] No `<InlineTOC>` in MDX (page shell provides TOC)
- [ ] Images use default MDX `img` (ImageZoom wired globally)

## Imports (when using compound components)

```mdx
import { Step, Steps } from "fumadocs-ui/components/steps";
import { Callout } from "fumadocs-ui/components/callout";
```

`Cards`, `Card`, `GeneratedReference`, and default MDX headings work without extra imports when registered in `@/components/mdx`.

## Section roots (`meta.json`)

- `use-erp`, `configure-tenant`, `operate-tenant`, `integrate` — reader IA
- `build-afenda` — engineer onboarding (legacy `(guides)` URLs redirect here)

Hub pages: Overview + `<Cards>` linking to subpages with **full-sentence** descriptions.

## Anti-patterns

| Avoid | Do instead |
|-------|------------|
| `# Same as frontmatter title` | `## Overview` + enriched prose |
| Thin index with only Cards | Overview paragraph + Cards |
| `description` as multi-sentence essay | Short metadata; put detail in Overview |
| Hand-editing `generated/` or sync stubs | Fix catalog / generator |
| Bold/marketing tone in every paragraph | Quiet body ink; weight in headings only |
