Your task is to create and harden the package:

```txt
packages/metadata
```

Package name:

```txt
@afenda/metadata
```

```txt
PKG-011
```

Architecture layer:

```txt
Metadata
```

Lifecycle:

```txt
Active
```


This package is **governance-only**. It must contain no React code, no UI components, no renderers, no CSS, no Tailwind, no business logic, no database access, no permission execution logic, and no runtime dependency on downstream packages.

The purpose of this package is to freeze the Afenda metadata architecture before `@afenda/metadata-ui` implementation begins.

Implement `@afenda/metadata` as the single source of truth for:

* metadata vocabulary
* surface definitions
* layout definitions
* section definitions
* renderer identity and compatibility rules
* registry lifecycle and governance rules
* presentation modes
* runtime context contracts
* metadata authority ownership map
* AI governance rules
* cross-package integration boundaries

The package must be stable, typed, serializable, testable, and safe for downstream consumption by `@afenda/metadata-ui`.

## 7. Required Authority Map


Must export:



```ts
may
mayNot
must
```

AI may:

* consume approved contracts from `@afenda/metadata`
* generate metadata schemas from approved governed arrays
* implement renderers only in `@afenda/metadata-ui`
* create tests for metadata governance

AI may not:

* invent new metadata authority domains
* invent new surface types outside `SURFACE_TYPES`
* invent new layout types outside `LAYOUT_TYPES`
* invent new section types outside `SECTION_TYPES`
* invent registry architecture outside `registryContract`
* invent runtime architecture outside `runtimeContract`
* merge `@afenda/metadata` with `@afenda/metadata-ui`
* add React, CSS, renderer, database, or business logic to `@afenda/metadata`

AI must:

* update contract version when authority changes
* update tests when governed arrays change
* update README when ownership changes
* keep metadata governance separate from metadata-ui implementation
